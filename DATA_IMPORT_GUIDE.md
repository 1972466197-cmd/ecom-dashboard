# 数据导入功能使用指南

**访问地址：** http://localhost:3000/import

### 核心功能

| 功能 | 说明 |
|------|------|
| 📥 Excel 导入 | 支持 .xlsx、.xls、.csv 格式 |
| ✏️ 手动填写 | 逐行填写销售数据 |

## 🎯 使用流程

### 方法一：Excel 导入

#### 1. 准备 Excel 文件

**支持的列名：**

| 中文列名 | 英文列名 | 说明 | 必填 |
|---------|---------|------|------|
| 店铺名称 | shop_name | 如"淘宝楠箐" | ✅ |
| 日期 | date | 格式：2026-03-23 | ✅ |
| 销售金额 | sales_amount | 销售额 | ✅ |
| 订单数 | order_count | 成交订单数 | ✅ |
| 商品数量 | item_count | 商品件数 | ❌ |
| 退款金额 | refund_amount | 退款金额 | ❌ |
| 退款订单数 | refund_order_count | 退款订单数 | ❌ |
| 推广费用 | ad_cost | 广告费/推广费 | ❌ |

**Excel 示例：**

```
| 店铺名称 | 日期       | 销售金额 | 订单数 | 商品数量 | 退款金额 | 推广费用 |
|---------|-----------|---------|-------|---------|---------|---------|
| 淘宝楠箐 | 2026-03-23 | 15000   | 150   | 200     | 500     | 5000    |
| 抖音楠箐 | 2026-03-23 | 20000   | 200   | 250     | 800     | 8000    |
```

#### 2. 导入步骤

1. 访问 http://localhost:3000/import
2. 点击"📥 Excel 导入"标签页
3. 点击虚线框或拖拽 Excel 文件到框内
4. 系统自动解析文件，显示预览（前 10 行）
5. 确认数据无误后，点击"✅ 确认导入"
6. 数据添加到待导入表格
7. 点击"💾 保存到 Supabase"
8. ✅ 显示绿色提示框："保存成功！成功：X, 失败：Y"
9. 数据自动保存到 Supabase

## 💾 Supabase 保存逻辑

### Upsert 去重

```typescript
// 同一天 + 同一个店铺自动覆盖更新
await supabase
  .from('sales_data')
  .upsert({
    shop_name: '淘宝楠箐',
    date: '2026-03-23',
    sales_amount: 15000,
    // ...其他字段
  }, {
    onConflict: 'shop_name,date' // 关键：去重约束
  })
```

**去重规则：**
- 如果 `shop_name='淘宝楠箐'` + `date='2026-03-23'` 已存在 → **覆盖更新**
- 如果不存在 → **创建新记录**

### 保存的数据表

**1. sales_data（销售数据）**
```sql
INSERT INTO sales_data (
  shop_name, date, sales_amount, 
  order_count, item_count, 
  refund_amount, refund_order_count
) VALUES (...)
```

**2. daily_marketing（推广数据）**
```sql
INSERT INTO daily_marketing (
  shop_name, date,
  taobao_ztc_spend, -- 淘宝店铺使用
  douyin_qc_spend   -- 抖音店铺使用
) VALUES (...)
```

## 🗑️ 清空数据说明

### 清空范围

**清空操作只影响：**
- ✅ 待导入表格中的数据
- ✅ 预览中的数据

**不影响：**
- ❌ Supabase 已保存的数据
- ❌ 数据库中的记录

### 清空确认

点击"🗑️ 清空数据"时：
1. 弹出确认对话框
2. 显示待清空的数据行数
3. 确认后清空待导入表格
4. 显示蓝色提示："已清空所有数据"

## ⚠️ 注意事项

### 1. Excel 格式要求

- ✅ 支持 .xlsx、.xls、.csv 格式
- ✅ 第一行必须是列名（表头）
- ✅ 日期格式：YYYY-MM-DD（如 2026-03-23）
- ✅ 数字格式：纯数字，不要带"¥"、","等符号

### 2. 店铺名称匹配

店铺名称用于自动识别平台：
- 包含"抖音" → 平台：douyin
- 包含"天猫"或"淘宝" → 平台：taobao
- 其他 → 平台：other

### 3. 数据去重

保存时使用 upsert 去重：
- 同一天 + 同一店铺的数据会覆盖更新
- 不会创建重复记录
- 如果需要同一天多条记录，请使用不同店铺名称

### 4. 网络要求

保存数据需要联网：
- 数据保存到 Supabase 云端
- 确保网络连接正常
- 保存失败会显示红色提示

## 🔧 常见问题

### Q1: Excel 导入后显示 0 行数据？

**原因：** 列名不匹配

**解决：** 检查 Excel 第一行是否包含以下列名：
- 店铺名称
- 日期
- 销售金额

### Q2: 保存失败，提示权限错误？

**原因：** Supabase RLS 策略未配置

**解决：**
```sql
ALTER TABLE sales_data ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow authenticated read" ON sales_data 
  FOR SELECT TO authenticated USING (TRUE);

CREATE POLICY "Allow authenticated write" ON sales_data 
  FOR ALL TO authenticated USING (TRUE);
```

### Q3: 数据重复保存？

**原因：** UNIQUE 约束未生效

**解决：** 检查数据库表结构：
```sql
ALTER TABLE sales_data 
ADD CONSTRAINT unique_shop_date UNIQUE(shop_name, date);
```

### Q4: 绿色提示框不显示？

**原因：** Toast 状态未正确更新

**解决：** 检查代码中是否正确调用 `showToast()`
