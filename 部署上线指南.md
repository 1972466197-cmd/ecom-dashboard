## 方法一：Vercel 部署（推荐，5 分钟上线）

### 第 1 步：打开 Vercel 官网

访问：https://vercel.com

### 第 2 步：注册/登录

- 点击 "Sign Up" 或 "Login"
- 推荐使用 **GitHub 账号登录**（最方便）
- 也可以用 Email、GitLab、Bitbucket 登录

### 第 3 步：导入项目

1. 点击 "**Add New Project**"
2. 选择 "**Import Git Repository**"
3. 选择你的 GitHub 账号
4. 找到并选择 `ecom-dashboard` 仓库
5. 点击 "**Import**"

### 第 4 步：配置项目

**Framework Preset:** Next.js（自动识别）  
**Root Directory:** `./`  
**Build Command:** `npm run build`  
**Output Directory:** `.next`

### 第 5 步：添加环境变量

点击 "**Environment Variables**"，添加以下变量：

```
NEXT_PUBLIC_SUPABASE_URL=https://你的项目.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的 anon public key
```

**如何获取 Supabase 密钥：**
1. 访问 https://app.supabase.com
2. 选择你的项目
3. 点击 "Settings" → "API"
4. 复制 "Project URL" 和 "anon public" key

### 第 6 步：点击 Deploy

点击 "**Deploy**" 按钮，等待 2-5 分钟

### 第 7 步：完成！

部署成功后，你会获得：
- **生产网址**: `https://your-project.vercel.app`
- **预览网址**: `https://your-project-git-main.vercel.app`

## 🗄️ Supabase 数据库配置

### 1. 执行数据库脚本

在 Supabase SQL Editor 中执行：
```
E:\山麓众创科技有限公司\ecom-dashboard\supabase-profit-schema.sql
```

### 2. 配置 RLS 权限

```sql
-- 启用 RLS
ALTER TABLE sales_data ENABLE ROW LEVEL SECURITY;
ALTER TABLE daily_marketing ENABLE ROW LEVEL SECURITY;
ALTER TABLE sku_costs ENABLE ROW LEVEL SECURITY;
ALTER TABLE profit_alerts ENABLE ROW LEVEL SECURITY;

-- 创建读取策略
CREATE POLICY "Allow public read" ON sales_data 
  FOR SELECT TO authenticated, anon USING (TRUE);

CREATE POLICY "Allow public write" ON sales_data 
  FOR ALL TO authenticated, anon USING (TRUE);

CREATE POLICY "Allow public read" ON daily_marketing 
  FOR SELECT TO authenticated, anon USING (TRUE);

CREATE POLICY "Allow public write" ON daily_marketing 
  FOR ALL TO authenticated, anon USING (TRUE);
```

### 3. 测试连接

在浏览器访问你的 Supabase 项目，确保可以正常连接。

## 🔧 遇到问题？

### 问题 1：构建失败

**错误：** Build failed

**解决：**
```bash
# 本地测试构建
npm run build

# 查看具体错误并修复
```

### 问题 2：环境变量不生效

**解决：**
1. 检查变量名是否正确（必须大写）
2. 检查是否以 `NEXT_PUBLIC_` 开头
3. 重新部署项目（Vercel 控制台 → Deployments → Redeploy）

### 问题 3：Supabase 连接失败

**解决：**
1. 检查 URL 是否正确（包含 https://）
2. 检查 Anon Key 是否正确
3. 检查 RLS 策略是否配置
4. 在 Supabase 控制台测试连接

### 问题 4：页面显示空白

**解决：**
1. 打开浏览器开发者工具（F12）
2. 查看 Console 错误
3. 检查 Supabase 连接是否正常
4. 检查环境变量是否配置
