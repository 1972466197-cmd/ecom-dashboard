# 数据持久化升级指南

## 🎯 本次升级内容

将 ecom-dashboard 从 localStorage 临时存储升级为 **Supabase 数据库持久化存储**，实现：

- ✅ 用户认证登录（Supabase Auth）
- ✅ 完整的数据库表结构（店铺、商品、销售数据）
- ✅ 数据导入功能（Excel → 数据库）
- ✅ 实时编辑保存（双击单元格 → 自动保存）
- ✅ 多用户权限支持（未来扩展）

---

## 📋 配置步骤

### 步骤 1：创建 Supabase 项目

1. 访问 https://app.supabase.com
2. 点击 **"New Project"**
3. 填写项目信息：
   - **Name**: 山麓众创科技 ERP
   - **Database Password**: 设置一个强密码（保存好！）
   - **Region**: 选择离你最近的（建议 Asia Pacific）
4. 等待项目创建完成（约 2-3 分钟）

---

### 步骤 2：获取 API 密钥

在 Supabase 项目后台：

1. 进入 **Settings** → **API**
2. 复制以下两个值：
   - **Project URL**: `https://xxxxxxxxxxxxx.supabase.co`
   - **anon public**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`

---

### 步骤 3：创建数据库表

1. 在 Supabase 后台进入 **SQL Editor**
2. 点击 **"New Query"**
3. 复制 `supabase-schema-full.sql` 文件的全部内容
4. 粘贴到 SQL Editor
5. 点击 **"Run"** 执行

✅ 成功后会看到：
- 8 张数据表（profiles, shop_groups, shops, products, product_shops, sales_data, orders, import_logs, settings）
- 初始化数据（4 个分组、28 个店铺、6 个商品示例）

---

### 步骤 4：配置环境变量

编辑项目根目录的 `.env.local` 文件：

```bash
NEXT_PUBLIC_SUPABASE_URL=https://你的项目 ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的 anon 密钥
```

> ⚠️ 注意：`.env.local` 不会被提交到 Git，安全存储

---

### 步骤 5：启用邮箱认证（可选）

如果需要用户注册登录：

1. 进入 **Authentication** → **Providers**
2. 确保 **Email** 已启用
3. （可选）配置自定义 SMTP 发送邮件

> 💡 开发模式可使用"快速登录"跳过认证

---

### 步骤 6：启动项目

```bash
cd ecom-dashboard
npm install
npm run dev
```

访问 http://localhost:3000

---

## 🚀 功能使用

### 1. 登录系统

访问 http://localhost:3000/login

**方式 A：快速登录（开发用）**
- 点击"快速登录（跳过认证）"
- 直接进入系统

**方式 B：正式登录**
- 先注册账号（需要邮箱验证）
- 使用邮箱 + 密码登录

---

### 2. 导入销售数据

1. 访问 **📥 数据导入** 页面
2. 选择店铺（如"大福珠宝"）
3. 上传 Excel 文件（支持 .xlsx, .xls）
4. 预览数据
5. 点击"开始导入"

**Excel 模板格式：**

| 日期 | 星期 | 支付金额 | 支付订单数 | 客单价 | 访客 | 全站推广 | 净利 | ... |
|------|------|----------|------------|--------|------|----------|------|-----|
| 2026-03-23 | 周一 | 125000 | 1389 | 89.9 | 3750 | 27500 | 31250 | ... |

> 💡 完整列名参考 `supabase-schema-full.sql` 中的 `sales_data` 表

---

### 3. 查看销售数据

访问 **🏪 店铺销售情况** 页面：

- 按分组查看（海林组、培君组、淑贞组、敏贞组）
- 展开店铺查看每日数据
- 双击数字可编辑（自动保存到数据库）
- 修改日期范围筛选

---

### 4. 商品管理

访问 **📦 商品管理** 页面：

- 查看商品列表
- 编辑商品信息
- 管理上下架状态
- 多平台上架管理

> ⚠️ 当前商品数据仍使用 localStorage，后续会升级到数据库

## 📊 数据库表结构

### 核心表

| 表名 | 说明 |
|------|------|
| `shop_groups` | 店铺分组（海林组、培君组等） |
| `shops` | 店铺信息（名称、平台、所属分组） |
| `products` | 商品信息（SKU、价格、成本、库存） |
| `product_shops` | 商品 - 店铺关联（多平台上架） |
| `sales_data` | 每日销售数据（核心业务数据） |
| `orders` | 订单明细（未来扩展） |
| `profiles` | 用户资料（扩展 Supabase Auth） |
| `import_logs` | 数据导入日志 |

## 🔐 安全说明

### Row Level Security (RLS)

已启用行级安全策略：

- ✅ 认证用户可读取所有业务数据
- ✅ 认证用户可插入/更新业务数据
- ✅ 用户资料只能访问自己的

### API 密钥安全

- `NEXT_PUBLIC_SUPABASE_ANON_KEY` 是公开密钥，可安全用于前端
- **Secret Key** 绝不能暴露在前端代码中
- 生产环境建议配置更严格的 RLS 策略

## 🛠 故障排查

### 问题 1：连接 Supabase 失败

**症状：** 页面显示"加载数据失败"

**解决：**
1. 检查 `.env.local` 配置是否正确
2. 确认 Supabase 项目状态正常
3. 查看浏览器控制台错误信息

---

### 问题 2：数据导入失败

**症状：** 导入时提示错误

**解决：**
1. 检查 Excel 列名是否匹配（参考模板）
2. 确认已选择店铺
3. 查看浏览器控制台和网络请求

---

### 问题 3：登录跳转循环

**症状：** 登录后又跳回登录页

**解决：**
1. 清除浏览器缓存和 localStorage
2. 检查 Supabase Auth 配置
3. 使用快速登录测试
