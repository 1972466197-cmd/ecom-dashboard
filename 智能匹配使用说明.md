# 🤖 智能表头匹配功能 - 使用说明

## ✅ 功能已完成

### 新增功能
在原有的 Excel 导入功能基础上，增加了**智能表头识别**功能：

1. **自动识别表头** - 上传 Excel 后自动识别表头并匹配数据库字段
2. **可视化匹配结果** - 显示哪些字段已识别，哪些未识别
3. **有关联自动导入** - 匹配的字段自动导入
4. **无关联默认跳过** - 未匹配的字段不会导入，保证数据准确

---

## 📊 支持的字段

系统自动识别以下字段（部分）：

### 必填字段
| 数据库字段 | 识别的 Excel 表头 |
|-----------|-----------------|
| date | 日期、时间、下单时间、付款时间、统计日期 |
| shop_name | 店铺名称、店铺、店名、主体名称 |
| pay_amount | 支付金额、销售额、成交金额、付款金额、总成交 |
| pay_orders | 支付订单数、订单数、付款订单数、支付买家数 |

### 可选字段
| 数据库字段 | 识别的 Excel 表头 |
|-----------|-----------------|
| visitors | 访客数、访客、UV |
| cart_count | 加购、加购件数 |
| ad_cost_total | 推广花费、全站推广、推广费、广告费 |
| shop_roi | 店铺 ROI、ROI、投产比、投入产出比 |
| refund_amount | 退款金额、成功退款金额 |
| gross_profit | 毛利、毛利润 |
| net_profit | 净利、净利润 |
| commission | 佣金、平台佣金 |
| logistics_fee | 物流费用、运费、快递费 |

**共支持 50+ 个字段的智能识别！**

## 🚀 使用步骤

### 1. 访问导入页面
访问：http://localhost:3000/import

### 2. 选择导入模板
根据你要导入的数据类型选择模板：
- 销售数据模板（30+ 列）
- 店铺日概况报表（17 列）
- 商品推广报表（56 列）
- 商品日概况报表（42 列）
- 刷单明细报表（42 列）
- 退款明细报表（54 列）

### 3. 选择店铺
从下拉列表中选择要导入的店铺

### 4. 上传 Excel 文件
点击上传区域，选择 Excel 文件（.xlsx 或 .xls）

### 5. 查看智能匹配结果 ⭐ 新增
上传后会自动显示表头识别结果：
- ✅ **绿色** = 已识别的字段（会导入）
- ⚠️ **灰色** = 未识别的字段（跳过不导入）

### 6. 开始导入
确认匹配结果后，点击"开始导入"按钮

## 📋 匹配示例

### 示例 1：标准表头
如果你的 Excel 表头是：
```
日期 | 支付金额 | 支付订单数 | 访客数 | 推广花费 | ROI
```
系统会识别：
- ✅ 日期 → date
- ✅ 支付金额 → pay_amount
- ✅ 支付订单数 → pay_orders
- ✅ 访客数 → visitors
- ✅ 推广花费 → ad_cost_total
- ✅ ROI → shop_roi

**识别率：6/6 = 100%**

### 示例 2：变体表头
如果你的 Excel 表头是：
```
交易时间 | 成交金额 | 订单数 | UV | 广告费 | 投产比
```
系统也会识别：
- ✅ 交易时间 → date
- ✅ 成交金额 → pay_amount
- ✅ 订单数 → pay_orders
- ✅ UV → visitors
- ✅ 广告费 → ad_cost_total
- ✅ 投产比 → shop_roi

**识别率：6/6 = 100%**

### 示例 3：混合表头
如果你的 Excel 表头是：
```
日期 | 支付金额 | 订单数 | 自定义字段 1 | 自定义字段 2
```
系统会识别：
- ✅ 日期 → date
- ✅ 支付金额 → pay_amount
- ✅ 订单数 → pay_orders
- ⚠️ 自定义字段 1 → 未识别（跳过）
- ⚠️ 自定义字段 2 → 未识别（跳过）

**识别率：3/5 = 60%**

## 💡 使用技巧

### 1. 提高识别率
使用标准表头名称可以获得 100% 识别率：
- 用"日期"而不是"时间戳"
- 用"支付金额"而不是"实际收款"
- 用"推广花费"而不是"开车费用"

### 2. 查看匹配结果
上传文件后，会立即显示匹配结果预览：
- 绿色字段 = 会导入的数据
- 灰色字段 = 跳过不导入的数据

### 3. 处理未识别字段
如果有重要字段未识别：
1. 修改 Excel 表头名称
2. 重新上传文件
3. 或者忽略（不影响其他字段导入）

### 4. 必填字段检查
系统会检查必填字段是否匹配：
- 日期（或统计日期）
- 支付金额（或成交金额）
- 支付订单数（或订单数）

如果必填字段未识别，会显示警告提示。

## ⚠️ 注意事项

### 1. 表头格式
- 表头必须在 Excel 的第一行
- 避免使用合并单元格
- 表头名称尽量简洁明确

### 2. 数据格式
- 日期格式：YYYY-MM-DD（如：2026-03-27）
- 金额格式：纯数字（如：15800，不要带¥符号）
- 避免空行和空列

### 3. 识别限制
- 系统只识别已知的字段类型
- 自定义字段不会导入
- 表头包含特殊字符可能影响识别

### 4. 数据准确性
- 未识别的字段不会导入，保证数据纯净
- 已识别的字段会自动转换数据类型
- 导入前请确认匹配结果

## 🔧 技术实现

### 智能匹配算法
1. **精确匹配** - 表头完全匹配已知字段
2. **包含匹配** - 表头包含关键字
3. **模糊匹配** - 基于语义相似度

### 字段配置
所有支持的字段配置在：
`src/lib/smart-match.ts`

### 扩展字段
如需添加新的字段识别，编辑 `FIELD_CONFIGS` 对象：

```typescript
export const FIELD_CONFIGS: Record<string, { labels: string[] }> = {
  'new_field': { 
    labels: ['新字段名', '别名 1', '别名 2'] 
  }
}
```
