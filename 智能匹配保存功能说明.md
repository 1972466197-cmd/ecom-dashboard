## 🎯 工作原理

### 之前的逻辑（硬编码）
```typescript
// 只能识别固定的表头名称
pay_amount: Number(row['支付金额'] || 0)
visitors: Number(row['访客'] || 0)
```

❌ **问题**：如果 Excel 表头是"成交金额"或"销售额"，就无法识别

### 现在的逻辑（智能匹配）
```typescript
// 自动识别多种表头变体
pay_amount: transformed.pay_amount || Number(row['支付金额'] || 0)
// ↑ 可以是：支付金额、成交金额、销售额、付款金额、总成交...
```

✅ **优势**：不管表头叫什么，都能自动识别并保存！

## 💡 技术实现

### 智能匹配函数
```typescript
const transformRowWithSmartMatch = (row, shopId, requiredFields) => {
  const transformed = { shop_id: shopId }
  
  // 遍历所有字段配置
  Object.entries(FIELD_CONFIGS).forEach(([dbField, config]) => {
    // 尝试所有可能的表头名称
    for (const label of config.labels) {
      if (row[label] !== undefined && row[label] !== '') {
        transformed[dbField] = convertValue(row[label], dbField)
        break
      }
    }
  })
  
  return transformed
}
```

### 字段配置
所有支持的字段配置在 `src/lib/smart-match.ts`：

## ⚠️ 注意事项

### 1. 数据保存
- ✅ 数据会保存到 Supabase 数据库
- ✅ 使用 upsert 方式，自动去重
- ✅ 基于唯一键（如 shop_id + date）避免重复

### 2. 必填字段
以下字段必须识别才能正常导入：
- **销售数据**：日期、支付金额、订单数
- **店铺日报**：统计日期、支付金额
- **商品日报**：日期、商品名称、支付金额

### 3. 未识别字段
- 未识别的字段会自动跳过
- 不影响其他字段的导入
- 可以在匹配结果预览中查看

### 4. 数据类型转换
- 数字自动去除逗号（1,234 → 1234）
- 日期自动格式化（2026-03-27）
- 空值自动处理为默认值
