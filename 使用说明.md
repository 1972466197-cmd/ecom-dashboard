## 🚀 快速启动

### 1. 打开命令行
```
Win + R → 输入 cmd → 回车
```

### 2. 进入项目目录
```cmd
cd /d E:\山麓众创科技有限公司\ecom-dashboard
```

### 3. 安装依赖（首次运行需要）
```cmd
npm install
```

### 4. 启动开发服务器
```cmd
npm run dev
```

### 5. 访问网站
浏览器打开：**http://localhost:3000**

## 📊 功能模块

### 1. 经营看板 (`/`)
- 总成交额、利润、ROI 汇总
- 各平台销售数据对比
- 数据可视化图表
- 支持导出 Excel

### 2. 商品管理 (`/products`)
- 商品列表展示
- 商品上下架管理
- 库存预警
- 商品搜索筛选

### 3. 店铺销售情况 (`/sales`)
- 每日销售数据表格
- 21 个核心字段
- 支持编辑和删除

### 4. 店铺分组管理 (`/shops`)
- 4 个小组：海林组、培君组、淑贞组、敏贞组
- 28 个店铺数据
- 支持编辑分组和店铺

### 5. 店铺每日数据 (`/shop-data/[店铺名]`)
- 每个店铺独立的每日数据表
- 支持导入/导出 Excel
- 支持编辑、删除、批量操作
- **数据保存到本地浏览器**

### 6. WPS 导入 (`/wps/import`)
- 支持 WPS 文档链接导入
- 支持本地 Excel 文件上传
- 数据预览和确认

### 7. 系统设置 (`/settings`)
- 店铺管理
- API 配置
- 通知设置

---

## 💾 数据存储

### 本地存储位置
数据保存在浏览器 localStorage 中：
- 每个店铺独立存储
- 键名格式：`shop_data_店铺名称`

### 数据同步
- 店铺数据页面保存后
- 经营看板自动更新汇总数据
- 刷新页面数据不丢失

## ⚠️ 注意事项

1. **数据保存**
   - 删除/编辑/添加数据后必须点击"💾 保存数据"
   - 数据保存在浏览器 localStorage
   - 清除浏览器数据会丢失

2. **浏览器要求**
   - 推荐使用 Chrome 或 Edge
   - 需要启用 localStorage

3. **端口占用**
   - 默认端口 3000
   - 如被占用会提示更换端口

4. **首次启动**
   - 首次运行需要安装依赖（约 2-5 分钟）
   - 确保已安装 Node.js

## 🛠 技术支持

### 常见问题

**Q: npm run dev 报错？**
A: 确保已安装 Node.js，运行 `npm install` 安装依赖

**Q: 数据丢失了？**
A: 检查是否清除了浏览器缓存，重新导入数据

**Q: 端口被占用？**
A: 关闭其他占用 3000 端口的程序，或修改 next.config.ts
